Hub Fortify integration

This simple parser allows for Fortify SSC 4.40 to parse Black Duck Hub security.csv files. The parsing allows for automating generation of Fortify SSC Issues. This is a proof-of-concept implementation leveraging some internal Fortify plugin architecture. Certain manual steps must be done as a one time setup and additional manual steps per process.

Build and Deployment

Building

This is a maven project, dependencies need Fortify jars that can be found in the /WEB-INF/lib directory within the Fortify tomcatforSSC folder.

  1. Open the pom.xml files and find the version of the fortify jars. They will be the 'fortify-public' and 'model' jars. They can be found in your Fortify installation in the ..\WEB-INF\lib folder.
  2. Install the fortify jars into your local repository.
  3. Run mvn package
  4. This will create a /target/ folder with the bds-fortify-integration.jar. It will also create a target/libs folder that will separate the provided and compile jars. The compile folder will contain a univocity-parsers-x.jar that needs to be placed on the Fortify class path.

Deploying

  1. Download the jar from the GitHub releases page.
  2. Edit the applicationContext.xml file located in ..\tomcatForSSC\webapps\ssc\WEB-INF\internal and add the following XML entry:
  3. (Add this right after the com.fortify entry)
  4. Copy the bds-fortify-integration-x.jar into the ..\WEB-INF\lib folder.
  5. Copy the univocity-parsers-x.jar into the same ..\WEB-INF\lib folder.
  6. Restart Fortify SSC.

Using the plugin

  1. Download the security.csv report from the Black Duck Hub.
  2. Rename the file to .log (Currently Fortify does not allow for CSV files and this is a temporary workaround)
  3. Navigate to your Application within the SSC user interface and select the scan tab.
  4. Upload the renamed security.log file
  5. Select the scan tab and you will find your scanned issues.
